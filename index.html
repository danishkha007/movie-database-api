üöÄ GitHub Pages Deployment

Step 1: Repository Setup

  1. Create a new GitHub repository
  2. Upload all files: index.html, style.css, app.js
  3. Upload your JSON data files: movies.json, persons.json, producers.json
  4. Go to repository Settings ‚Üí Pages
  5. Select "Deploy from a branch" and choose "main" branch

Step 2: Data File Requirements

  1. Ensure all JSON files are valid JSON format
  2. Files must be accessible via HTTP (same domain as HTML)
  3. For GitHub Pages, files must be committed to the repository
  4. Test locally using a development server (not file:// protocol)

Step 3: Using the API

Once deployed, your API will be accessible at:

  • GitHub Pages: https://username.github.io/repository-name/#/api/movies
  • Custom Domain: https://your-domain.com/#/api/movies

Note: This is a client-side API simulation that loads data from JSON files. Changes won't persist unless you update the source files.
